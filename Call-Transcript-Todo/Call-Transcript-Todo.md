# 通话录音转写

收到音频文件后，完成三件事：转录音频 → 生成飞书文档 → 提取待办并确认。

---

## Step 1: 安装依赖并转录

首次使用时安装 faster-whisper（后续跳过）：

```bash
pip install faster-whisper --break-system-packages
```

转录脚本：

```python
from faster_whisper import WhisperModel

model = WhisperModel('small', device='cpu', compute_type='int8')
segments, info = model.transcribe('<音频文件路径>', language='zh')

transcript = []
for seg in segments:
    transcript.append(seg.text)

full_text = '\n'.join(transcript)
```

转录完成后，将原始文本保存备用，进入 Step 2。

**注意事项：**

- faster-whisper 首次加载模型会下载约 500MB，需要等待
- 如果音频是电话录音（8kHz 采样率），whisper small 模型足够使用
- 如果转录质量不理想，可尝试 `medium` 模型（更慢但更准）

---

## Step 2: 整理实录与生成纪要

拿到转录原文后，对内容做两层加工：

### 2.1 整理实录

将转录原文整理为可读的对话实录：

- 根据语义和上下文判断说话人，标记为 A / B（如有更多参与者则 C / D...）
- 合并同一说话人的连续短句
- 修正明显的语音识别错误（如专有名词、人名）
- 格式：每段以 `**A：**` 或 `**B：**` 开头

### 2.2 生成纪要

用连贯段落概括通话内容，包含：

- 通话背景：谁和谁、什么事由
- 主要讨论内容和关键结论
- 后续安排和跟进事项

纪要控制在 3-5 句话，简洁精炼，不用列表。

### 2.3 提取待办

从通话内容中识别具体的待办事项：

- 明确承诺要做的事
- 约定的时间节点或截止日期
- 需要跟进或确认的事项

如果通话中没有明确的待办事项，写"本次通话未提取到明确的待办事项。"

---

## Step 3: 生成飞书文档

使用 `feishu_doc action=write` 创建文档，内容结构如下：

```markdown
# 通话录音转写

**来源**：[文件名，或从文件名中提取的来电号码/姓名]
**时间**：[转录日期]
**时长**：[如可获取]
**语言**：中文

---

## 实录

**A：** [说话内容]

**B：** [说话内容]

---

## 纪要

[连贯段落，3-5句话概括]

---

## 待办

[待办事项列表，或"本次通话未提取到明确的待办事项。"]
```

**必须用 `feishu_doc action=write`**，这样 markdown 标题会正确转换为飞书文档的标题格式。不要用 `update_block`。

---

## Step 4: 发送结果

在聊天中发送飞书文档链接和待办摘要，格式如下：

```
📄 [飞书文档链接]

待办事项：
1. [待办内容]
2. [待办内容]

确认计入工作待办？(是/否/修改)
```

如果没有待办事项，只发文档链接即可：

```
📄 [飞书文档链接]

本次通话未提取到明确的待办事项。
```

---

## Step 5: 待办确认与写入

用户确认后才将待办写入 `memory/todo.md`：

- 用户回复"是"/"确认"/"OK" → 写入 todo.md，回复"已计入工作待办"
- 用户回复"否"/"不要" → 不写入，回复"已忽略"
- 用户回复修改内容 → 按修改调整后再次确认

未收到确认前，不写入 todo.md。

---

## 文件名解析

通话录音文件名通常包含有用信息，尽量提取：

| 文件名模式                           | 提取内容                |
| ------------------------------- | ------------------- |
| `XX_20260323.m4a`               | 来源：刘总，时间：2026-03-23 |
| `Record_20260323_143022.mp3`    | 时间：2026-03-23 14:30 |
| `CallRecord_+8613912345678.m4a` | 来源：+8613912345678   |
| 其他格式                            | 用文件名原文作为来源          |

## 示例

**用户发送**：音频文件 `XX_20260323.m4a`

**处理流程**：

1. faster-whisper 转录 → 获得中文文本
2. 整理实录（A/B 对话格式）+ 生成纪要 + 提取待办
3. `feishu_doc action=write` 创建飞书文档
4. 聊天发送：

```
📄 https://feishu.cn/docx/XXX

待办事项：
1. 下周三前发送合作方案初稿给XX
2. 协调技术团队安排下周现场演示

确认计入工作待办？(是/否/修改)
```
